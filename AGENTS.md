# Directra — Agent 视频制作平台

## 项目概述

Directra 是一个**通用 Agent 视频制作平台**，不是 AI 一键生成工具。

核心定位：**自然语言入口 + Skill 驱动的领域工作流 + 导演台调度 + 长视频一致性系统 + 在线剪辑台**

目标是完成"可控的视频项目"，而非"生成一个视频"。

---

## 系统分层架构

| 层级 | 职责 |
|------|------|
| 第一层：自然语言入口 | 把一句话解析为项目意图（类型/受众/时长/风格/领域） |
| 第二层：导演台/调度层 | 规划整个制作流程，决定阶段顺序和用户介入点 |
| 第三层：Skill 层 | 领域化规则与工作流（知识讲解/品牌营销/儿童故事等） |
| 第四层：生成层 | 调用各类模型产出脚本、分镜、素材 |
| 第五层：一致性层 | 维护角色/场景/故事/风格/音频状态 |
| 第六层：编辑层 | 人工精修、局部重做、时间轴导出 |

---

## 核心模块说明

### 导演台（Director Agent）
- 调度中心，不是聊天助手
- 职责链：意图分析 → Skill 路由 → 规划 → 脚本 → 分镜 → 素材 → 一致性检查 → 视频拼接 → 编辑器接管 → 导出
- 模式：**Agent 主导流程，用户主导决策**

### Skill 系统
- Skill = **领域知识 + 工作流 + 约束 + 风格 + 资产绑定**
- 不是 Prompt 模板，而是领域生产协议
- 每个 Skill 定义：任务拆解方式、脚本写法、分镜规则、必须确认节点、一致性要求、优先工具

内置 Skill 示例：
- 知识讲解
- 品牌营销
- 儿童故事
- 课程视频
- 口播 + B-roll
- 新闻解读

### 长视频一致性系统
- 结构：**Act → Scene → Shot → Clip**
- 每个 Clip 独立生成，通过一致性上下文控制连续性
- 维护状态：角色状态、场景状态、故事状态、风格状态、音频状态
- 衔接机制：首尾帧、桥接镜头、转场

### 编辑器
- 平台最终落点，不是附属功能
- 支持：局部修改、局部重做、锁定、转场、字幕、音乐替换、版本快速变体
- Chat 和 Timeline 互相映射（自然语言控制时间轴）

## 核心开发原则

1. **做项目系统，不做 Prompt 工具** — 入口是项目意图理解，不是 Prompt 输入框
2. **Skill 是领域工作流，不是模板** — Skill 决定做法，平台提供能力
3. **Agent 是调度系统，不是聊天机器人** — 导演台总控，串联各层
4. **长视频靠组织，不靠一次生成** — 多短片段 + 一致性维护 + 拼接
5. **编辑器是落点，不是补充** — 生成初稿后用户在编辑器接管
6. **支持局部修改，不强迫全部重来** — 某段/某镜头可独立修改或锁定

---

## 用户交互模式

- 先出方向，用户确认
- 先出大纲，用户确认
- 先出分镜，用户确认
- 某段不满意 → 只改某段
- 某个镜头不满意 → 只重做该镜头
- 自然语言直接控制编辑器（"第三个镜头更紧张一点"）

---

## 技术决策约定

- 使用 Codex API（`@anthropic-ai/sdk`）作为 Agent 核心
- 多 Agent 编排时，导演台 Agent 作为顶层调度者
- 一致性层独立于生成层，作为独立服务/模块存在
- Skill 应可动态加载/扩展，不硬编码在主流程中
