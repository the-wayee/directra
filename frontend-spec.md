# Directra — 前端产品规格文档（v0 开发用）

> 本文档专为 v0 快速原型开发而写，描述每个页面的布局、组件、交互行为与视觉风格。
> 所有数据均为 mock，不依赖后端。

---

## 产品背景

### Directra 是什么

Directra 是一个**面向短视频创作者的 Agent 视频制作平台**。

它不是 AI 一键生成视频的工具，而是一个**可控的视频项目系统**。核心定位：

> 把一个模糊的视频想法，组织成一个可推进、可确认、可修改、可最终交付的视频项目。

### 用户是谁

**ToC 产品**，目标用户是**短视频创作者**——有内容想法，但缺乏制作能力和结构化组织能力的人。他们不是视频专业人士，也不是企业用户，是有创作欲望的个人内容创作者。

### 核心工作流

用户进入 Directra 不是在写 prompt，而是在**发起一个视频制作项目**。系统的工作方式是：

1. 用户用一句话描述想做什么视频
2. 系统（导演台 Agent）理解意图，匹配领域工作流（Skill）
3. 分阶段推进：方向确认 → 大纲确认 → 脚本确认 → 分镜确认
4. 每个关键节点暂停，等用户确认后继续
5. 初稿完成后用户进入 Studio 编辑器精修
6. 最终导出

核心原则：**Agent 主导流程，用户主导决策**。

### 双模式设计

产品有两个互补的工作模式：

- **Agent 创作模式**：从 0 到 1，通过对话把模糊想法组织成结构化项目（脚本、分镜）
- **Studio 编辑模式**：精修阶段，时间轴编辑器 + 自然语言指令双轨控制

两个模式共享同一个项目，可以随时切换，不是两个独立产品。

### Skill 是什么

Skill = 领域化的生产协议（不是 prompt 模板）。

内置 Skill 示例：知识讲解 / 品牌营销 / 儿童故事 / 课程视频 / 口播+B-roll / 新闻解读

不同 Skill 决定脚本结构、分镜规则、确认节点，让系统针对不同内容类型按正确方式生产。

### 局部修改是核心能力

用户**不需要全部重来**：
- 某段脚本不满意 → 只改那段
- 某个镜头不满意 → 只重做那个 Shot
- Studio 里锁定满意的部分，只改其他的

## 设计基调

**产品感觉**：专业创作工具，不是消费级 AI 玩具。参考 Linear、Cursor、Vercel Dashboard 的设计语言。

**颜色**：深色主题（dark mode 优先）。背景 `#0A0A0A` 或 `#111111`，卡片/面板 `#1A1A1A`，边框 `#2A2A2A`，主色调用低饱和度的紫色或青色作为强调色。

**字体**：Inter（UI 文字）+ Mono 字体（代码/脚本内容）

**圆角**：适中，8-12px。卡片不要太圆，工具感。

**动效**：克制，快速。过渡 150-200ms。流式输出文字要有光标闪烁动效。

### 顶部 NavBar

### 项目卡片（ProjectCard）

卡片尺寸：宽约 280px，比例 16:9 封面 + 底部信息区。

**封面区（上 2/3）**：
- 深色背景占位，中间显示项目名称首字母大图
- 右上角：Skill badge（知识讲解 / 品牌营销 / 儿童故事...），小药丸形状，半透明背景
- 右上角另有"..."菜单（hover 时出现）：重命名 / 复制 / 删除

**信息区（下 1/3）**：
- 第一行：项目名称（加粗，14px）
- 第二行：状态 badge + 最后更新时间（两端对齐）

**状态 badge 颜色规则**：
- `● 等待确认` → 橙黄色，有轻微脉冲动画
- `⚙ 生成中` → 蓝色，有旋转图标
- `✓ 编辑中` → 绿色
- `○ 草稿` → 灰色
- `↗ 已导出` → 紫色

**交互**：
- hover 时卡片整体轻微上浮（translateY -2px），边框高亮
- 点击进入 `/projects/[id]`

### 空状态

无项目时，主内容区居中显示：
- 大图标（导演板或视频图标）
- 标题："开始你的第一个视频项目"

## 页面 3：Agent 创作模式 `/projects/[id]`

### 整体布局

**左右双栏**，左 Chat，右内容预览。顶部工具栏。

```
┌──────────────────────────────────────────────────────────────────┐
│ ← 项目列表   量子纠缠科普视频   知识讲解        [进入 Studio ↗]    │  ← TopBar 56px
├───────────────────────┬──────────────────────────────────────────┤
│                       │                                          │
│   Chat Panel          │   内容预览区                              │
│   （左侧，固定宽度      │   （右侧，可滚动）                         │
│    约 380-420px）      │                                          │
│                       │                                          │
│   [消息列表]           │   [阶段 Tab 或进度条]                     │
│                       │                                          │
│                       │   [当前阶段内容]                          │
│                       │                                          │
│                       │                                          │
│   ─────────────────── │                                          │
│   [输入框]  [发送]     │                                          │
└───────────────────────┴──────────────────────────────────────────┘
```

### TopBar

- 左：`← 项目列表`（面包屑）→ 项目名（可点击编辑重命名）→ Skill badge
- 右：阶段进度指示 + `[进入 Studio ↗]` 按钮（当项目有分镜后激活）

### 左侧：Chat Panel

**消息气泡样式**：
- 用户消息：右对齐，背景用主色调，圆角气泡
- Agent 消息：左对齐，无气泡背景，直接渲染 Markdown（更像 Cursor 风格）
- Agent 头像：小型图标（导演板图标），紧贴消息左侧

**流式输出**：
- Agent 文字逐字出现，末尾有光标闪烁 `|`
- 输出完毕后光标消失

**确认节点卡片（关键交互）**：

当 Agent 需要用户确认时，消息流中出现特殊卡片：

```
┌───────────────────────────────────────────┐
│  📋  请确认项目大纲                         │
│                                           │
│  大纲已在右侧展示，请浏览后确认是否继续。     │
│                                           │
│  [✓ 确认，继续生成]   [✎ 我想修改]          │
└───────────────────────────────────────────┘
```

- 卡片有轻微边框，背景稍亮于消息区
- 两个按钮：主操作（确认）+ 次操作（修改）
- 点击"确认"：卡片变为已确认状态（打勾，灰色），Agent 继续推进
- 点击"修改"：输入框获得焦点，用户可以输入修改意见

**输入区**：
- 底部固定，分隔线上方
- textarea（可换行），右侧发送按钮
- placeholder："告诉 Directra 你的想法..."
- 支持 `Enter` 发送，`Shift+Enter` 换行

### 右侧：内容预览区

顶部是**阶段进度条**（或 Tab 切换）：

```
 Brief  →  大纲  →  脚本  →  分镜  →  素材  →  时间轴
  ✓          ✓       当前      ○        ○         ○
```

每个阶段圆点 + 连线，已完成为实心绿色，当前为主色高亮，未开始为灰色。

**各阶段内容展示**：

#### Brief（项目概要）
简洁的卡片，列出字段，有"修改"图标（铅笔）可以 inline 编辑每个字段。

#### 大纲（Outline）
树状/列表结构，显示 Act / Scene 层级：
```
Act 1  引入概念（1:30）
  └─ Scene 1.1  什么是量子
  └─ Scene 1.2  纠缠从何而来
Act 2  核心解释（2:00）
  └─ Scene 2.1  EPR 悖论
  └─ Scene 2.2  贝尔不等式
Act 3  总结与展望（1:30）
```
每个条目右侧有"修改"/ "锁定"图标。锁定后显示锁图标，表示该部分不会被重生成。

#### 脚本（Script）
分段展示每个 Scene 的脚本文本：
```
Scene 1.1 — 什么是量子

[口播]
想象你有一枚硬币。在你观察它之前，它既可能是正面，
也可能是反面...

[画面提示] 硬币旋转动画，模糊化 → 叠加态示意

─────────────────────────────────────
                    [修改这段] [锁定]
```

#### 分镜（Storyboard）
Shot 卡片网格（2列或3列）：

```
┌─────────────┐  ┌─────────────┐
│             │  │             │
│  [图片占位]  │  │  [图片占位]  │
│             │  │             │
├─────────────┤  ├─────────────┤
│ Shot 1  3s  │  │ Shot 2  4s  │
│ 全景，实验室 │  │ 特写，粒子   │
│ "想象你有一  │  │ "在量子世界  │
│  枚硬币..."  │  │  里..."      │
│ [重做][锁定] │  │ [重做][锁定] │
└─────────────┘  └─────────────┘
```

Shot 卡片元素：
- 上方：16:9 图片占位区（灰色渐变，中间有相机图标）
- 下方：Shot 编号 + 时长、镜头描述（2行）、台词（2行，截断+tooltip）
- 底部操作：`[重新生成]` `[锁定]`，hover 时出现 `[删除]`

全局操作区（分镜视图顶部）：
- 左：视图切换（网格 / 列表）
- 右：`[全部重新生成]` `[进入 Studio 编辑]`

### TopBar

- 左：`← 返回创作模式`（返回不丢失数据）+ 项目名
- 右：`[保存]`（手动保存，或有自动保存提示）+ `[导出▾]`（下拉：导出MP4 / 导出分镜 PDF）+ undo/redo 图标

### 左侧：素材面板

Shot 列表，竖向排列：
- 每个条目：Shot 缩略图（小）+ Shot 名称 + 时长
- 当前选中：高亮背景
- 锁定状态：右侧锁图标 🔒（灰色，不可拖动）
- 拖拽排序（未锁定的 Shot）
- 右键菜单：重命名 / 重新生成 / 锁定 / 删除 / 复制

### 中间：预览窗口

- 16:9 黑色预览区，视频播放占位
- 底部播放控制：`◀◀` `▶` `▶▶` + 时间码 + 全屏按钮
- 选中时间轴某片段时，预览跳到对应位置

### 右侧：属性面板

根据选中的时间轴片段动态显示：

**视频轨道片段选中时**：
- 基本信息：Shot 编号、时长（可编辑）、镜头类型
- 台词文本（可编辑 textarea）
- 操作按钮：`[重新生成此片段]` `[替换素材]` `[锁定/解锁]`

**字幕片段选中时**：
- 字幕文字（可编辑）
- 时间范围（开始/结束，可调）
- 字体/大小/颜色（简化选项）

**音乐轨道选中时**：
- 当前音乐名称
- `[更换音乐]` 按钮
- 音量（slider）

### 时间轴区域

**轨道**：
- 视频轨（Video）：显示 Shot 片段，不同 Shot 用不同深浅色区分
- 字幕轨（Subtitle）：细长条，显示字幕时间段
- 音乐轨（Music）：全长波形占位条

**片段视觉**：
- 每个片段内显示 Shot 名（如"Shot 3"）
- 锁定片段有锁图标叠加，背景颜色略不同（带斜纹）
- 片段边缘可拖拽调整时长（暂时是视觉占位，不需要真实功能）

**时间刻度**：顶部显示时间轴（00:00, 00:05, 00:10...）

**播放指针**：红色竖线，可拖动（视觉占位）

### 底部：Chat Bar

- 默认展开，高度约 60px
- 右侧 `[▲收起]` 按钮，收起后只留一个细条（带 `[展开聊天]`）
- 展开状态显示最近几条 AI 编辑指令记录（类似命令历史）
- 输入自然语言编辑指令，例如：
  - "第三个镜头时长缩短1秒"
  - "全程背景音乐换成轻松的钢琴曲"
  - "Shot 5 的台词改得更简短一点"

## 通用组件规范

### Loading 状态

**文字流式输出**：逐字出现 + 光标闪烁（`|`），不用 spinner。

**内容块骨架**：Skeleton 占位，灰色渐变条（shimmer 动画）。

**按钮 loading**：按钮内 spinner（小圆圈）+ 禁用状态，不要改变按钮尺寸。

### 状态 Badge 组件

```
● 等待确认   → 背景 #2A1F00，文字 #F59E0B，小圆点脉冲
⚙ 生成中    → 背景 #001F3A，文字 #3B82F6，旋转图标
✓ 编辑中    → 背景 #002A12，文字 #22C55E
○ 草稿      → 背景 #1A1A1A，文字 #6B7280
↗ 已导出    → 背景 #1A0030，文字 #A855F7
```

### Skill Badge

药丸形状，`font-size: 11px`，边框 + 低透明度背景：
- 知识讲解 → 蓝色系
- 品牌营销 → 橙色系
- 儿童故事 → 粉色系
- 课程视频 → 绿色系
- 口播 + B-roll → 青色系

### 确认节点卡片

用于 Chat 中的用户确认节点，视觉上要比普通消息更醒目：
- 左侧有主色调竖线（4px 宽）
- 背景略亮，有细边框
- 底部操作区有明确的主/次按钮

## 交互细节补充

### 局部修改流程（关键体验）

用户在分镜视图中点击某个 Shot 的 `[重新生成]`：

1. 该 Shot 卡片进入"生成中"状态（spinner + 文字变灰）
2. 其他 Shot 正常显示，不受影响
3. 约 1.5s 后（mock 延迟），新内容刷新进卡片，带 fade-in 动画
4. Chat 区自动出现一条消息："Shot 3 已重新生成，你可以继续修改或确认"

### 锁定/解锁流程

- 点击 `[锁定]`：图标变为 🔒，背景加浅色斜纹，操作按钮隐藏
- 锁定后"重新生成全部"之类的批量操作会跳过锁定项
- 点击 🔒 图标解锁

### 双模式切换

- Agent 创作模式 → Studio：点击右上角 `[进入 Studio ↗]`，过渡动画（淡出淡入），不丢失任何数据
- Studio → Agent 创作模式：点击左上角 `← 返回创作模式`，直接切换

### Chat 与右侧内容联动

- 当 Agent 说"请确认大纲"时，右侧自动切换到"大纲"Tab
- 用户点击右侧某个 Shot 的"修改"→ Chat 区自动出现"你想如何修改 Shot X？"的引导

## v0 提示建议

以下是给 v0 的补充说明：

- **整体深色主题**，不要亮色
- 尽量用 shadcn/ui 组件（Button、Card、Badge、Tabs、Textarea、Separator）
- 使用 Tailwind 控制间距和颜色，颜色用 `zinc`/`neutral` 系列做灰色层级
- 强调色建议用 `violet` 或 `blue`（选一种，统一全局）
- 所有 loading 动画使用 CSS 动画，不用第三方动画库
- 字体大小规范：标题 16-18px，正文 14px，标签/Badge 11-12px
- 先把**首页 + 新建项目流程 + Agent 创作模式**跑通，Studio 可以后做
