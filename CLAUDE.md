# Directra — Agent 视频制作平台

## 项目概述

Directra 是一个**通用 Agent 视频制作平台**，不是 AI 一键生成工具。

核心定位：**自然语言入口 + Skill 驱动的领域工作流 + 导演台调度 + 长视频一致性系统 + 在线剪辑台**

目标是完成"可控的视频项目"，而非"生成一个视频"。

---

## 系统分层架构

| 层级 | 职责 |
|------|------|
| 第一层：自然语言入口 | 把一句话解析为项目意图（类型/受众/时长/风格/领域） |
| 第二层：导演台/调度层 | 规划整个制作流程，决定阶段顺序和用户介入点 |
| 第三层：Skill 层 | 领域化规则与工作流（知识讲解/品牌营销/儿童故事等） |
| 第四层：生成层 | 调用各类模型产出脚本、分镜、素材 |
| 第五层：一致性层 | 维护角色/场景/故事/风格/音频状态 |
| 第六层：编辑层 | 人工精修、局部重做、时间轴导出 |

---

## 核心模块说明

### 导演台（Director Agent）
- 调度中心，不是聊天助手
- 职责链：意图分析 → Skill 路由 → 规划 → 脚本 → 分镜 → 素材 → 一致性检查 → 视频拼接 → 编辑器接管 → 导出
- 模式：**Agent 主导流程，用户主导决策**

### Skill 系统
- Skill = **领域知识 + 工作流 + 约束 + 风格 + 资产绑定**
- 不是 Prompt 模板，而是领域生产协议
- 每个 Skill 定义：任务拆解方式、脚本写法、分镜规则、必须确认节点、一致性要求、优先工具

内置 Skill 示例：
- 知识讲解
- 品牌营销
- 儿童故事
- 课程视频
- 口播 + B-roll
- 新闻解读

### 长视频一致性系统
- 结构：**Act → Scene → Shot → Clip**
- 每个 Clip 独立生成，通过一致性上下文控制连续性
- 维护状态：角色状态、场景状态、故事状态、风格状态、音频状态
- 衔接机制：首尾帧、桥接镜头、转场

### 编辑器
- 平台最终落点，不是附属功能
- 支持：局部修改、局部重做、锁定、转场、字幕、音乐替换、版本快速变体
- Chat 和 Timeline 互相映射（自然语言控制时间轴）

## 核心开发原则

1. **做项目系统，不做 Prompt 工具** — 入口是项目意图理解，不是 Prompt 输入框
2. **Skill 是领域工作流，不是模板** — Skill 决定做法，平台提供能力
3. **Agent 是调度系统，不是聊天机器人** — 导演台总控，串联各层
4. **长视频靠组织，不靠一次生成** — 多短片段 + 一致性维护 + 拼接
5. **编辑器是落点，不是补充** — 生成初稿后用户在编辑器接管
6. **支持局部修改，不强迫全部重来** — 某段/某镜头可独立修改或锁定

---

## 用户交互模式

- 先出方向，用户确认
- 先出大纲，用户确认
- 先出分镜，用户确认
- 某段不满意 → 只改某段
- 某个镜头不满意 → 只重做该镜头
- 自然语言直接控制编辑器（"第三个镜头更紧张一点"）

## 技术决策约定

- **前端优先开发策略**：先跑通 UI 和交互，后端用 mock/假数据代替，明确产品形态后再接真实后端
- **先文字后生成**：Agent 对话 → 文字 Brief/大纲/脚本/分镜描述（用户可编辑）→ 用户确认后才触发图片/视频生成。生成异步进行，不阻塞交互
- Skill 应可动态加载/扩展，不硬编码在主流程中
- 一致性层独立于生成层，作为独立服务/模块存在

## 系统分层（三层架构）

```
┌─────────────────────────────────────┐
│  前端 Next.js (React + App Router)   │  UI 渲染、用户交互、SSE 流式展示
└────────────────┬────────────────────┘
                 │ Server Actions / API Routes
┌────────────────▼────────────────────┐
│  Node BFF (Next.js 服务端)           │  职责：
│                                     │  · Auth & Session 管理
│                                     │  · 数据库读写（Prisma）
│                                     │  · SSE 流式中转（前端 ↔ Python）
│                                     │  · 轻量业务逻辑、路由调度
│                                     │  · 文件上传协调（presigned URL）
└────────────────┬────────────────────┘
                 │ HTTP / SSE
┌────────────────▼────────────────────┐
│  Python 执行层 (FastAPI)             │  职责：
│                                     │  · Claude Agent 编排（多轮对话、Skill 路由）
│                                     │  · 文字内容生成（脚本、分镜描述）
│                                     │  · 图片/视频生成任务（异步队列）
│                                     │  · 一致性引擎
│                                     │  · 消息队列任务执行
└─────────────────────────────────────┘
```

**各层职责原则**：
- Node 层：擅长 I/O 密集、流式推送、Auth、ORM；不做 AI 计算
- Python 层：擅长 AI SDK 调用、科学计算、生成任务；不管 Web 会话
- 两层通过 HTTP + SSE 通信，Python 层可独立扩容

## 技术栈（现代化选型，2025）

### 前端 + Node BFF
| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | Next.js 15+ (App Router) | Server Components + Server Actions + API Routes |
| UI 组件 | shadcn/ui v4 | 组件直接 copy 进项目，完全可控 |
| 样式 | Tailwind CSS v4 | |
| 表单 | React Hook Form + Zod | 表单验证 + 类型推导 |
| 服务端状态 | TanStack Query v5 | 数据请求、缓存、同步 |
| 客户端状态 | Zustand | 轻量，仅用于纯客户端 UI 状态 |
| AI 流式输出 | Vercel AI SDK v5 | SSE 流式渲染，中转 Python 层的 stream |
| Auth | Better Auth | 功能最完整的开源方案，支持 2FA/passkey/RBAC |
| 数据库 ORM | Prisma | schema.prisma 管理 DDL，`prisma migrate dev` 快速迭代 |
| 数据库 | PostgreSQL | |

### Python 执行层
| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | FastAPI + Uvicorn | 异步、高性能，原生支持 SSE streaming |
| AI Agent | Anthropic Python SDK | Claude 多轮对话、tool_use 编排 |
| 消息队列 | Dramatiq + Redis | 异步生成任务（图片/视频），比 Celery 更现代 |
| 缓存 | Redis | 任务状态、会话缓存 |
| 文件存储 | S3 presigned URL | 素材直传，本地开发用 MinIO |

> **数据库归属**：PostgreSQL 由 Node 层（Prisma）统一管理迁移。Python 层通过 asyncpg 直连读写，不重复管理 schema。

## 核心数据模型

```
Project（工作区/主题）
  ├── Conversation 1 → 对话/视频 A（messages[]）
  ├── Conversation 2 → 对话/视频 B（messages[]）
  └── 未来：SharedContext（角色、风格、品牌资产共享）
```

- **Project**：工作区，包含多个对话。字段：title/status/skillId
- **Conversation**：一次 AI 对话，产出一个视频。字段：title/phase（CHAT→BRIEF→...→EDIT）
- **Message**：对话消息。字段：role/type/content + 确认节点字段
- 路由：对话页 `/c/[conversationId]`，侧边栏按 Project 的最新 Conversation 展示

## 产品定位

- **ToC 产品**，面向**短视频创作者**
- 核心用户痛点：有内容想法，但缺乏制作能力和结构化组织能力
- 交互设计原则：**轻量确认、快速出稿感、低门槛入口**
- 不是企业工具，不做复杂团队协作功能
